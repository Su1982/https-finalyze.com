## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do NOT** open a public GitHub issue for security vulnerabilities.
2. Send a private report via GitHub's [Security Advisories](../../security/advisories/new).
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
4. You will receive a response within **48 hours**.
5. Once the vulnerability is confirmed and fixed, a public disclosure will be made.

## Security Best Practices

- Always use the latest version of this project
- Keep your dependencies up to date
- Never commit sensitive data (passwords, API keys, tokens) to the repository
